# Contributing to Zaryx Studios

¡Gracias por tu interés en contribuir! Favor de leer las pautas antes de abrir issues o pull requests.

## Cómo reportar un bug o solicitar una feature
Usa las plantillas disponibles cuando crees un issue.

## Estilo de código
Mantén el mismo estilo que el proyecto al que contribuyes; revisa `/.editorconfig` o las guías internas de cada repo.

## Pull Requests
1. Crea una rama (`feature/nombre` o `bugfix/nombre`).
2. Asegúrate de que los tests pasen y que el código esté formateado.
3. Abre un pull request describiendo el cambio.
4. Los mantenedores revisarán y solicitarán cambios o harán merge.

## Comunicación
Únete a nuestro [Discord](http://discord.zaryxstudios.dev/) para discusiones en tiempo real.
